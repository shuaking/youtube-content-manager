# Phase 3 补充应用功能分析

## 概述
Language Reactor 提供了多个补充应用来增强语言学习体验。这些应用都需要登录才能使用完整功能。

## 1. PhrasePump (⛽)

### 功能描述
- 主要是听力练习工具
- 系统选择包含已保存单词的句子
- 同时引入新词汇
- 适合初学者和高级学习者

### 界面布局（未登录状态）
- 居中的登录提示卡片
- 说明文字："请登录以使用此免费功能。以便我们可以保存您的进度"
- 蓝色"登录"按钮
- 底部功能说明文字

### 设计特点
- 简洁的居中布局
- 深色背景
- 卡片式设计
- 清晰的行动号召（CTA）

## 2. Chatbot - Aria (🤖)

### 功能描述
- AI 对话伙伴
- 提供即时纠正
- 探索无限话题
- 帮助建立语言自信

### 界面布局
- 顶部标签栏（当前只有 "Aria" 标签）
- 工具栏按钮（导出、全屏等）
- 主聊天区域
  - 欢迎消息："嗨，我是 Aria，一个虚拟对话伙伴。"
  - 语言提示："如果你不知道用英语说什么，你可以用中文（简体）写。"
  - 话题建议卡片：
    - 著名人物
    - 心理学家
    - 游戏
    - 情况
    - 只是咏叹调
    - 词汇故事
- 底部输入区域
  - 多行文本框："您的消息"
  - 录音按钮
  - 快捷键提示："按 [ESC] 切换快捷键/输入模式"

### 设计特点
- 类似现代聊天应用的界面
- 话题卡片采用网格布局
- 深色主题
- 清晰的输入区域分隔

## 3. 已保存项目 (Saved Items)

### 功能描述
- 词汇管理中心
- 查看和管理保存的单词和短语
- 支持导出到 Anki

### 界面布局（未登录状态）
- 顶部标签栏：
  - "所有单词"
  - "标记的词语"（当前选中）
  - "保存的短语"
- 右上角"设置"按钮（带下拉菜单）
- 工具栏按钮
- 居中登录提示："请登录以使用此功能。"
- 蓝色"登录"按钮

### 设计特点
- 标签式导航
- 三个主要分类
- 设置按钮提供额外配置选项

## 4. 文本/我的媒体 (Text/My Media)

### 功能描述
- 导入网页内容
- 上传书籍
- 配合文字转语音学习

### 界面布局（未登录状态）
- 顶部多标签导航：
  - YouTube
  - Netflix
  - 图书
  - FSI/DLI
  - 媒体文件 NEW
  - 播客
  - 我的文本（当前选中）
  - 学习资源
- 居中登录提示
- 蓝色"登录"按钮

### 设计特点
- 8 个标签的宽标签栏
- 统一的登录门控界面
- 与其他页面一致的设计语言

## 通用设计模式

### 1. 导航结构
- 固定顶部导航栏
- 包含：媒体内容、Chatbot、PhrasePump、已保存项目、帮助、设置、论坛、登录
- 语言选择器（显示当前学习语言）

### 2. 登录门控
- 所有主要功能都需要登录
- 统一的登录提示界面
- 简短的功能说明
- 突出的"登录"按钮

### 3. 标签式界面
- 多个应用使用标签导航
- 标签栏位于页面顶部
- 当前标签有视觉高亮

### 4. 色彩方案
- 深色背景（接近黑色）
- 蓝色作为主要强调色（按钮、链接）
- 白色/浅灰色文字
- 卡片使用略浅的背景色

### 5. 布局原则
- 居中对齐的主要内容
- 充足的留白
- 卡片式设计元素
- 响应式布局

## 实现优先级建议

### 高优先级
1. **Chatbot (Aria)** - 核心交互功能，用户参与度高
2. **已保存项目** - 词汇管理是学习的核心

### 中优先级
3. **PhrasePump** - 听力练习工具
4. **文本导入** - 扩展学习材料

### 实现策略
- 先实现基础 UI 框架和布局
- 使用模拟数据展示功能
- 添加"需要登录"的占位状态
- 逐步添加实际功能逻辑

## 技术要点

### 组件需求
- TabNavigation 组件（可复用）
- LoginGate 组件（登录提示卡片）
- ChatInterface 组件（聊天界面）
- TopicCard 组件（话题建议卡片）
- VocabularyList 组件（词汇列表）
- AudioRecorder 组件（录音功能）

### 状态管理
- 用户登录状态
- 保存的词汇数据
- 聊天历史
- 学习进度

### API 集成点
- AI 聊天接口（可使用 OpenAI API 或类似服务）
- 语音识别/合成
- 词汇数据存储
- 用户进度跟踪
